# ARSÈNE — боевая сборка магазина

React (Vite) + **Supabase** (база, авторизация, хранилище фото).
Оплата: **СБП** + оформление через Telegram-менеджера **@pinxty**.

---

## Что уже настроено в коде

- Товары, заказы, аккаунты и настройки сайта живут в **Supabase** — общие для всех устройств.
- Регистрация и вход — настоящая авторизация Supabase (пароли хранит Supabase, не мы).
- Права админа определяются полем `role='admin'` в таблице `profiles` (никаких паролей в коде).
- Фото товара автоматически сжимается до **900×900** и загружается в Storage.
- Остатки товара уменьшаются автоматически при создании заказа (триггер в БД).
- При оформлении заказа открывается Telegram **@pinxty** с готовым текстом заказа.

## Шаг 1. Подготовить Supabase (если ещё не сделано)

1. В проекте Supabase открыть **SQL Editor → New query**.
2. Вставить содержимое `supabase_schema.sql` и нажать **Run**.
3. **Storage → New bucket** → имя `product-images` → отметить **Public bucket**.
4. **Authentication → Providers → Email** — включён. На время запуска удобно выключить
   подтверждение почты: **Confirm email = OFF**.

## Шаг 3. Сделать себя администратором

1. На сайте нажать иконку профиля → **Регистрация** → создать аккаунт на свою почту.
2. В Supabase → **SQL Editor** выполнить (подставив свою почту):

```sql

set role = 'admin'
where id = (select id from auth.users where email = 'ВАША_ПОЧТА');
```

3. Перезайти на сайте. Появится вкладка **Админ** — там товары, заказы и настройки сайта.

## Шаг 4. Наполнить магазин

В админке: **Добавить товар** → название, бренд, категория, тип, цена, размеры, цвета,
состав, **остаток** и **сроки доставки**, фото (до 6, первое — главное).

## Шаг 5. Выложить в интернет (Vercel)

```bash

git commit -m "ARSENE store"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/arsene-store.git
git push -u origin main
```

1. На **vercel.com** войти через GitHub → **Add New → Project** → выбрать репозиторий.

у регистратора. HTTPS включится автоматически.

## Как проходит заказ

1. Покупатель наполняет корзину и нажимает **Оформить заказ**.
2. Заполняет контакты, доставку (Курьер / СДЭК / Самовывоз) и выбирает оплату
   (**СБП онлайн** или при получении).
3. По кнопке **«Оформить и написать менеджеру»**:
   - заказ сохраняется в базу со статусом **«Обработка»**;
   - открывается Telegram **@pinxty** с готовым сообщением: номер заказа, товары,
     размеры, суммы, доставка, реквизиты оплаты, имя и телефон.
4. В админке (вкладка **Заказы**) вы видите заказ целиком, **одобряете** или отклоняете,
   меняете статус: Обработка → Подтверждён → Собирается → В доставке → Доставлен.
5. Покупатель видит статус в разделе **Мои заказы**.

## Безопасность: что важно знать

- `anon` ключ — **публичный**, его можно держать в коде и в Vercel. Так и задумано.
- **`service_role` ключ никогда никому не показывайте** и не кладите во фронтенд.
- Доступ к данным ограничен политиками RLS: товары читают все, меняет только админ;
  заказы видит их владелец или админ; статус меняет только админ.
- Оплата по СБП подтверждается менеджером вручную. Автоматическое подтверждение оплаты
  (эквайринг) потребует ИП/ООО и платёжного шлюза — это следующий этап, если понадобится.

## Если что-то не работает

- **«Не удалось подключиться к базе»** — проверьте `.env`: URL без `/rest/v1/`, ключ целиком.
- **Фото не загружаются** — не создан bucket `product-images` или он не публичный.
- **Не видно вкладку «Админ»** — не выполнен Шаг 3 (роль `admin`), либо нужно перезайти.
- **После регистрации просит подтвердить почту** — выключите Confirm email (Шаг 1.4)
  или подтвердите письмо.
